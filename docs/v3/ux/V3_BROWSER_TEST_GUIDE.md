## 1. 필수 시나리오 체크리스트

### A. 일일 추천 상태 배너 (DayStatusBanner)

#### A-1. PRE_1535 시나리오
**조건**: 현재 시간이 15:35 KST 이전

**확인 사항**:
- [ ] 배너가 화면 최상단에 표시됨
- [ ] 헤더 문구: "기존 추천을 유지하고 있습니다"
- [ ] 보조 문구: "추천은 15:35에 확정됩니다" (또는 유사한 문구)
- [ ] 배너 색상/아이콘 적절함

**스크린샷**: (캡처 필요)

#### A-2. POST_1535 + 신규 추천 있음
**조건**: 현재 시간이 15:35 KST 이후 + 오늘 신규 추천 생성됨

**확인 사항**:
- [ ] 배너 헤더: "오늘 장후 시간외 매수를 위한 추천이 확정되었습니다" (또는 유사)
- [ ] 보조 문구: "정규장 종가 기준으로 반영되었습니다" (또는 유사)
- [ ] 배너가 PRE_1535와 다른 색상/톤

**스크린샷**: (캡처 필요)

#### A-3. POST_1535 + 신규 추천 없음 + 기존 유지
**조건**: 15:35 이후 + 오늘 신규 없음 + 기존 ACTIVE 존재

**확인 사항**:
- [ ] 배너 헤더: "오늘은 기존 추천을 유지합니다"
- [ ] 보조 문구: "추천 기준에 변화가 없어 새 추천은 생성되지 않았습니다" (또는 유사)

**스크린샷**: (캡처 필요)

#### A-4. POST_1535 + 추천 없음
**조건**: 15:35 이후 + ACTIVE/WEAK_WARNING 없음

**확인 사항**:
- [ ] 배너 헤더: "오늘은 새로운 추천이 없습니다"
- [ ] 보조 문구: "조건이 충족되면 추천으로 안내드립니다" (또는 유사)

**스크린샷**: (캡처 필요)

#### A-5. HOLIDAY (시장 휴장일)
**조건**: 주말 또는 공휴일

**확인 사항**:
- [ ] 배너 헤더: "오늘은 시장 휴장일입니다"
- [ ] 보조 문구: "추천 상태는 다음 거래일에 갱신됩니다" (또는 유사)
- [ ] 기존 ACTIVE/WEAK_WARNING 카드 유지됨

**스크린샷**: (캡처 필요)

### B. NEW 요약 카드 (DailyDigestCard)

- [ ] 종목명 나열 없음
- [ ] CTA 버튼 없음
- [ ] 강조/색상 과도하지 않음

**스크린샷**: (캡처 필요)

#### B-2. has_changes = false
**조건**: 오늘 변화 없음

**확인 사항**:
- [ ] NEW 요약 카드가 표시되지 않음
- [ ] 배너 바로 아래가 ACTIVE 영역 또는 다른 섹션

**스크린샷**: (캡처 필요)

### C. 현재 추천 영역 (ACTIVE / WEAK_WARNING)

#### C-1. 기본 상태
**확인 사항**:
- [ ] 섹션 상단 설명 문구: "아래는 손절 조건에 도달하지 않아 관찰 중인 추천 목록입니다."
- [ ] ACTIVE와 WEAK_WARNING 카드가 섞여 표시됨
- [ ] 기본 접힘 상태 (최근 12개만 표시)
- [ ] "관찰 중인 추천 더 보기" 버튼 있음 (숫자 없음)
- [ ] "n개 중 m개" 같은 숫자 요약 문구 없음

**스크린샷**: (캡처 필요)

#### C-2. ACTIVE 카드 상세
**확인 사항**:
- [ ] 종목명 표시
- [ ] 뱃지: "추천 유효"
- [ ] 요약: "손절 조건에 도달하지 않은 상태입니다"
- [ ] 메타: "추천일 YYYY.MM.DD · 경과 N거래일" (둘 다 표시)
- [ ] 가이드: "관찰 중인 추천입니다"
- [ ] 수익률이 보조 정보로만 표시 (작은 폰트, 강조 없음)
- [ ] 매수/진입/기회 같은 행동 유도 문구 없음

**스크린샷**: (ACTIVE 카드 확대 캡처 필요)

#### C-3. WEAK_WARNING 카드 상세
**확인 사항**:
- [ ] 뱃지: "흐름 약화" (ACTIVE와 색상/톤 구분)
- [ ] 요약: "추천 당시 흐름이 약해지고 있습니다"
- [ ] 가이드: "변동성 확대에 유의하세요"
- [ ] ACTIVE와 시각적으로 구분됨

**스크린샷**: (WEAK_WARNING 카드 확대 캡처 필요)

#### C-4. 접기/펼치기 동작
**확인 사항**:
- [ ] "관찰 중인 추천 더 보기" 클릭 → 전체 목록 펼침
- [ ] 펼친 후 "접기" 버튼 표시 (숫자 없음)
- [ ] 접기 클릭 → 다시 12개만 표시
- [ ] 동작이 자연스러움

**스크린샷**: (펼침 상태 캡처 필요)

cd backend
python -m uvicorn main:app --reload --port 8010

cd frontend
npm run dev
