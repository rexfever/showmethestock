# V3 메인 화면 브라우저 테스트 체크리스트

## 테스트 환경
- 브라우저: Chrome, Safari, Firefox (최신 버전)
- 화면 크기: 모바일 (375px), 태블릿 (768px), 데스크톱 (1920px)

## 1. 일일 추천 상태 배너 (DayStatusBanner)

### ✅ 테스트 케이스
- [ ] **dailyDigest가 null일 때**: 기본 배너("기존 추천을 유지하고 있습니다") 표시
- [ ] **PRE_1535**: "기존 추천을 유지하고 있습니다. 추천은 15:35에 확정됩니다." 표시
- [ ] **POST_1535 + 신규 추천**: "오늘 장후 시간외 매수를 위한 추천이 확정되었습니다." 표시
- [ ] **POST_1535 + 기존 유지**: "오늘은 기존 추천을 유지합니다." 표시
- [ ] **POST_1535 + 추천 없음**: "오늘은 새로운 추천이 없습니다." 표시
- [ ] **HOLIDAY**: "오늘은 시장 휴장일입니다." 표시
- [ ] 배너가 항상 최상단에 표시됨
- [ ] 배너가 1개만 표시됨

## 2. NEW 요약 카드 (DailyDigestCard)

### ✅ 테스트 케이스
- [ ] **has_changes = true일 때**: 카드 표시
- [ ] **has_changes = false일 때**: 카드 숨김
- [ ] **신규 추천만 있을 때**: "• 신규 추천 N건"만 표시
- [ ] **관리 필요만 있을 때**: "• 관리 필요 N건"만 표시
- [ ] **관리 종료만 있을 때**: "• 관리 종료 N건"만 표시
- [ ] **모든 변화가 있을 때**: 모든 항목 표시
- [ ] **모든 값이 0일 때**: 카드 숨김
- [ ] 종목명이 나열되지 않음
- [ ] 강조/CTA 버튼이 없음

## 3. 현재 추천 영역 (ACTIVE/WEAK_WARNING)

### ✅ 테스트 케이스
- [ ] 섹션 상단 설명 문구 표시: "아래는 손절 조건에 도달하지 않아 관찰 중인 추천 목록입니다."
- [ ] 기본 접힘 상태 (12개만 표시)
- [ ] "관찰 중인 추천 더 보기" 버튼 표시
- [ ] 펼치기 후 모든 항목 표시
- [ ] "접기" 버튼 표시
- [ ] ACTIVE 카드 문구 확인:
  - 뱃지: "추천 유효"
  - 요약: "손절 조건에 도달하지 않은 상태입니다"
  - 가이드: "관찰 중인 추천입니다"
  - 메타: "추천일 YYYY.MM.DD · 경과 N거래일"
- [ ] WEAK_WARNING 카드 문구 확인:
  - 뱃지: "흐름 약화"
  - 요약: "추천 당시 흐름이 약해지고 있습니다"
  - 가이드: "변동성 확대에 유의하세요"
- [ ] 수익률이 보조 정보로 표시됨 (강조되지 않음)
- [ ] 숫자 요약 문구가 없음 ("총 N개", "N개 중" 등)

## 4. 관리 필요 영역 (BROKEN)

### ✅ 테스트 케이스
- [ ] 별도 섹션으로 표시됨 (메인 추천과 분리)
- [ ] 기본 접힘 상태
- [ ] 최대 5개만 표시
- [ ] 최신 BROKEN 우선 정렬
- [ ] 카드 문구 확인:
  - 뱃지: "관리 필요"
  - 요약: "추천 당시 가정이 깨졌습니다"
  - 가이드: "리스크 관점에서 정리를 고려하세요"
  - 보조: "현재 흐름은 추천 기준과 다르게 전개되고 있습니다"
- [ ] 수익률/가격/숫자 표시 없음
- [ ] CTA 버튼 없음
- [ ] 0개일 때: "현재 관리 필요 종목이 없습니다" 표시

## 5. 관찰 종료된 추천 (ARCHIVED)

### ✅ 테스트 케이스
- [ ] 별도 섹션으로 표시됨
- [ ] 기본 접힘 상태
- [ ] 섹션 상단 설명: "아래는 일정 기간 관찰 후 추천 관리가 종료된 기록입니다."
- [ ] 카드 문구 확인:
  - 뱃지: "관찰 종료"
  - 요약: "추천 관리 기간이 종료되었습니다"
  - 보조: "성과와 무관하게 추천 관찰이 마무리되었습니다"
  - 메타: "추천일 YYYY.MM.DD · 관찰 N거래일"
- [ ] archive_return_pct 사용 (ARCHIVED 전환 시점의 수익률)
- [ ] 수익률이 낮은 시각적 우선순위로 표시됨
- [ ] CTA 버튼 없음
- [ ] 실패처럼 보이지 않음 (중립 색상)

## 6. 에러 처리

### ✅ 테스트 케이스
- [ ] API 오류 시 에러 메시지 표시
- [ ] dailyDigest가 없어도 배너 표시 (기본값)
- [ ] dailyDigest 필드가 undefined여도 안전하게 처리
- [ ] 네트워크 오류 시 재시도 버튼 표시
